About Us

Flye

Flye is a GalaxyTrakr tool for assembling long read sequences, such as the outputs of Oxford Nanopore Technologies or PacBio instruments.

Inputs: fastq or fasta file(s) from a long read sequencing method. NOTE:Though Flye is designed to run on sequences of mediocre quality, if you have reason to believe your sequencing data is of questionable quality, it would be wise to run sequences through a filtering tool like filtlong first. See tutorial for filtlong here.

Outputs, with those that are most important for most of our audience highlighted:

  • fasta assembly file(s) that can be used in further analyses, such as Nanoplot.
  • tabular data file(s) containing data about the contigs in the sequence, such as their length, their coverage, and whether or not they repeat.
  • graph_dot file(s) containing assembly data that can be downloaded and visualized in software, browser, or command line tool versions of Graphviz.
  • gfa1 file(s) containing assembly data that can be downloaded and visualized in any tool that can handle this file format, such as ABySS, SPAdes, GfaViz, etc.

This tutorial assumes that you have already imported fastq or fasta file(s) from a long read sequencer into a GalaxyTrakr history. If you need assistance with that task, please see this tutorial.

Search for and click on Flye in the tool search bar of GalaxyTrakr.

You have the option of running Flye on an individual sequencing file (simply click on only one file in the list of files when the multiple sheet icon is selected), multiple sequencing files that have not previously been a collection (select the icon with multiple sheets), and a collection of sequencing files (for instructions on how to make such a collection, visit this tutorial. Note that when the multiple datasets option is used, each sequence to be assembled must be selected as if selecting multiple files in Windows (hold down the ctrl key and click on items in the list individually or hold down the shift key and highlight the last item in a group to be selected).

Select a mode depending on which type of instrument your sequences came from.

You will next see the Number of polishing iterations prompt. GenomeTrakr colleagues currently recommend 3 polishing iterations.

No other changes from the defaults are currently recommended. If you believe your use case would benefit from further changes to the defaults and woud like to discuss with subject matter experts, please email us at microseqtrain@fda.hhs.gov.

Decide whether you would like to Generate a log file and toggle that option accordingly.

Choose whether you would like to be emailed when filtlong is finished running using the toggle, and click Execute to run the tool.
