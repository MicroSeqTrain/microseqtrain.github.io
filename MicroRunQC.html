Skesa

Skesa is a tool for creating an assembly from fastq or fasta files that have breaks in the repeat sections of the genome.

Coming soon!

For the purposes of this tutorial, it is assumed that the user has previously created a history, uploaded fastq or fasta files, and created a paired list of them in GalaxyTrakr. If you have not already done so and would like to learn more about those steps, please visit this tutorial for fastq import and this tutorial for creating a collection of fastq pairs.

Once you have your data in your history, search for “skesa” in the “search tools” bar and select the “skesa” tool. For the purposes of this tutorial, we will be using basic skesa, not SkesaMLST.

Skesa gives you the option of starting with single FASTQ or FASTA files, forward and reverse FASTQ files, and paired FASTQ files as your inputs.

If your sequencing method outputs single fastq files, you may select the “Single FASTQ” option under the “Select Input” dropdown menu. This will enable the options of selecting a single dataset, multiple datasets, or a dataset collection.

If you have uploaded forward and reverse files for your samples but have not paired them, select the “Forward and Reverse FASTQ” option from the Select Input dropdown menu. You will then be able to input a single dataset, multiple datasets, or a collection of datasets from the resulting dropdown menus. Ensure that you have a forward and a reverse file for each of your samples.

If you have created a collection of FASTQ pairs to use as your input, select this option from the “Select Input” dropdown menu. Then, select the name of the step in which you have created this collection under the menu that says “Paired FASTQ.”

In this tutorial, we will only be covering the “Basic” options for skesa. Select that option from the Options Type dropdown menu. If you are using a large dataset, you may want to switch the “Email notification” toggle on so that you receive an email notification if you navigate away from the GalaxyTrakr page while working on other tasks.

When you are satisfied with your input and options selections, click “Execute.”

After the tool completes, you will see the output in the History pane of the history in which you ran skesa. The output will be in fasta format and can be used in your future analyses by referencing their step in the history. The fasta output takes the form of a list where each item is individually downloadable by clicking on the hyperlink of the fasta file’s name and then on the floppy disk that appears.

Documentation for Skesa can be found here on the developer's GitHub page.
