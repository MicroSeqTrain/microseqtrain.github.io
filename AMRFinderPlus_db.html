About Us

AMRFinderPlus_db

Amrfinderplus_db is a tool for detecting genes for virulence factors and resistance to antibiotics, heat, acid, metal, and biocides in bacterial genomes. The

Inputs: fasta files or proteins or both together, if you have a gff file that will facilitate this.

Outputs: text file(s) containing alleles associated with antibiotic resistance (and other resistance genes and virulence factors, if you have enabled the "plus" genes option) and their locations within the input genomes. Note:Genes marked "plus" in the output have not necessarily been validated for phenotype. If you see an AMR gene in your results, particularly if it is marked as a "plus" gene, the bacterium may not not necessarily exhibit the associated antimicrobial phenotype.


Once you have your data in your history, search for amrfinderplus_db in the search tools bar and select the amrfinderplus_db tool.

For our purposes, most often, the input will be nucleotide FASTA files. You have the option of running AMRFinderPlus on an individual fasta file (select single dog-eared sheet symbol), multiple fasta files that have not previously been a collection (select the icon with multiple sheets), and a collection of fasta files (for instructions on how to make such a collection, visit this tutorial. Note that when the multiple datasets option is used, each sequence to be filtered must be selected as if selecting multiple files in Windows (hold down the ctrl key and click on items in the list individually or hold down the shift key and highlight the last item in a group to be selected).

Choose the database you would like to use to detect genes of interest. Generally, you will choose the most recent database. An exception to this may be if you have used an older version of a database previously and would like to remain consistent with versioning. If you have already run the analysis for new data with a different version of the database, it is a good practice to return to your old data and run AMRFinderPlus_db on it again using the new database.

Other options that are available to you for adjusting the tool’s outputs are:

  • Selecting an organism of interest to filter results
  • Selecting a threshold for alleles needed to determine presence of a gene
  • Minimum acceptable coverage of reference protein
  • Whether or not to add plus genes to the report. Toggle this to the on position if you would like to see resistance or virulence genes other than those for antibiotics
  • Suppress proteins common to a taxonomy group, if you have selected organism-specific results and would like to eliminate outputs common to the taxonomy group
  • Report all target positions of mutations
  • Only toggle the next step about format of protein identifiers if you have uploaded protein FASTA files as well as nucleotide

Choose whether you would like to be emailed when amrfinderplus_db is finished running using the toggle, and click Execute to run the tool.
