NanoPlot

NanoPlot is a GalaxyTrakr tool for visualizing sequencing data and alignments resulting from Oxford Nanopore Technologies (ONT) long read sequences.

Inputs: fastq, fasta, fastq_rich, fastq_minimal, summary, bam, or cram file(s) from ONT or subsequent sequence processing. NOTE:If you have reason to believe your sequencing data is of questionable quality, it would be wise to run sequences through a filtering tool, like filtlong, first. See tutorial for filtlong here.

Outputs:

  • Text NanoStats (pre-filtering) file containing summary statistics about the quality of the file(s) analyzed.
  • Text NanoStats post filtering file containing summary statistics about the quality of the file(s) analyzed after they have been filtered.
  • png or svg file(s) containing a histogram visualizing read length data.
  • png or svg file(s) containing a histogram visualizing read length data after it has undergone a log transformation.
  • An html file that contains all of the summary data and plots generated by the tool.

This tutorial assumes that you have already imported fastq or fasta file(s) from a long read sequencer into a GalaxyTrakr history. If you need assistance with that task, please see this tutorial.

Search for and click on NanoPlot in the tool search bar of GalaxyTrakr.

You have the option of running NanoPlot on an individual sequencing file (select single dog-eared sheet symbol), multiple sequencing files that have not previously been a collection (select the icon with multiple sheets), and a collection of sequencing files (for instructions on how to make such a collection, visit this tutorial. Note that when the multiple datasets option is used, each sequence to be analyzed must be selected as if selecting multiple files in Windows (hold down the ctrl key and click on items in the list individually or hold down the shift key and highlight the last item in a group to be selected).

Note:Unlike some other tools, NanoPlot will give you the option of whether to run the tool on multiple files with their results appearing in separate data files and plots (batch) or with the results in a single file for each output type (combined) in the Select multifile mode dropdown menu.

It is not necessary to change the modifiers under Options for filtering or transforming input prior to plotting. However, you can use these options to dictate how NanoPlot filters data prior to assessing quality. Email us at if you would like to discuss how these options may be beneficial in your particular use case. There is also a toggle under this section that should be switched if the tool will be analyzing bam files.

There is also an optional section for changing the appearance of the graphs NanoPlot produces and other output attributes, Options for customizing the plots created. Attributes that can be changed include the color of the plots, the bivariate format(s) of the plots, and whether or not N50 is marked on the read length histogram. Note:If you would like to choose svg format instead of the default png format for the histograms NanoPlot produces, the Specify the output format of the plots. dropdown in this section is where you would do it.

Choose whether you would like to be emailed when NanoPlot is finished running using the toggle, and click Execute to run the tool.
