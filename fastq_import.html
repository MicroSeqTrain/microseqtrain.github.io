GalaxyTrakr Basics: FASTQ Import

These tutorials will show you two ways of importing genomic data into GalaxyTrakr, from your computer/network or from NCBI. For the purposes of this tutorial, it is assumed that you have created a History into which your files and subsequent jobs will go. If you need help doing this, please visit this tutorial.


If you are importing your data from your local network or your own computer, please perfom the following:

  1. After sigining in to GalaxyTrakr, make sure you are in the History where you would like your files uploaded then click on the button that reads Upload Data.
  2. Click Choose local files and navigate to the folder that contains your files of interest, select those files, and click Open. The file names will now appear in the pane in GalaxyTrakr, along with their size and options for their import. We would suggest leaving Type as the Auto-detect default.

    You do not need to select an organism type under the Genome column unless you are certain that the sequence you are uploading is from a pure sample and you know its ID. If you have selected a file for upload by mistake, remove it from the upload by clicking on the trashcan icon to the right of it.

  3. Once you have your settings the way you want them, click Start to begin the upload. The Status bars for the files will begin filling. The pane does not have to remain open for the upload to continue. Click the Close button to return to the GalaxyTrakr main page. The progress of the upload can also be seen in the same Upload data button that was used to initiate the upload. Click that button again if you need to open that pane again. The upload can be paused by doing so and clicking the Pause button.

The files will appear in the History once the upload is complete.


If you are importing data from NCBI without using your computer as an intermediary, one option is to use the Faster Download and Extract Reads in FASTQ tool. It will pull FASTQ files from NCBI and deposit them in your history. The files will be divided into collections depending on whether they are paired- or single-end data, the first and second collections created, or a different type of file, which will be placed in a third Other data collection. A text file log will also be created containing a simple breakdown of spots and reads read and reads written for each NCBI sequencing run record (SRR) accession.

  1. After signing in to GalaxyTrakr, make sure you are in the history where you would like your files uploaded, then search for NCBI in the Tool search bar at the left-hand side of the main GalaxyTrakr page. You are looking for the tool called Faster Download and Extract Reads in FASTQ in the list of tools typing NCBI gives you. Click on it. You have three options for input in this tool:

    • SRR accession, where you can input a single SRR that will have its FASTQ files added to the History.
    • List of SRA accession, one per line, where you can input list of SRRs that will have their FASTQ files added to the History. Please make sure these are in line-by-line format rather than a comma separated list.
    • SRA archive in current history, where an SRR accession list in a file uploaded to your History will be converted into a list of FASTQ files. Please see the documentation at the bottom of the tool's page in GalaxyTrakr for further details of this method.
  2. Advanced options in the tool will not apply in most use cases. Please contact us at microseqtrain@fda.hhs.gov if you would like to discuss use of those options.
  3. Toggle on Email notification at the bottom of the page, if you would like GalaxyTrakr to email you when the data has been completely extracted from NCBI.
  4. When the options are as you wish, click Execute to begin the extraction from NCBI.
